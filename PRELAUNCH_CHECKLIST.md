# Финальная проверка перед запуском

## Что уже защищено

- Автоматизация выполняется на сервере и продолжает работать, когда сайт и браузер закрыты.
- API-токены Wildberries шифруются AES-256-GCM и не возвращаются в браузер.
- Настройки API и добавление юрлиц доступны только администратору.
- Для каждого сотрудника можно задать список доступных юрлиц через `legal_entity_ids`.
- Выбор текущего юрлица хранится отдельно в каждом браузере и больше не переключает кабинет у других сотрудников.
- Сервер проверяет доступ к юрлицу при чтении статистики, изменении правил, ручной синхронизации и запуске проверки.
- Экспорт, почасовая статистика и аналитика ограничены выбранным юрлицом.
- Есть защита от повторного пополнения, блокировка параллельной обработки кампании и суточные лимиты по московскому времени.
- Вход временно блокируется после 10 неверных попыток с одного адреса за 15 минут.
- Контейнер запускается без root-прав, без Linux capabilities и доступен приложению только через локальный порт сервера.

## Обязательные действия перед боевым запуском

1. Подключить домен и HTTPS. Не передавать логины, пароли и токены через обычный HTTP.
2. После подключения HTTPS заменить адрес сервера в расширении и его `host_permissions`, собрать новый ZIP и переустановить расширение на компьютерах.
3. Убедиться, что файл `/opt/wb-autofund/.env` имеет права `600`, а каталог с данными и резервные копии доступны только администратору сервера.
4. Для каждого сотрудника использовать отдельный логин и уникальный пароль длиной не менее 12 символов.
5. В `ADDITIONAL_USERS_JSON` явно указать `legal_entity_ids`. Если поле пропущено, сотрудник увидит все текущие и будущие юрлица.
6. Проверить резервное копирование Docker volume `wb_autofund_data`.
7. Сначала оставить `WB_LIVE_DEPOSITS`, `WB_LIVE_RESUME` и `WB_LIVE_PAUSE` выключенными, проверить журнал, затем включать действия по одному.

## Как кабинеты отображаются пользователям

- Администратор видит все юрлица, может добавлять и менять API-токены.
- Сотрудник видит только юрлица, перечисленные в его `legal_entity_ids`.
- Новое юрлицо сразу появляется у администратора.
- Сотрудник не увидит новое юрлицо, пока его ID не будет добавлен в `legal_entity_ids` и контейнер не будет перезапущен.
- Кампании, журнал, почасовая статистика и аналитика всегда относятся к выбранному юрлицу.
- Каждый компьютер и каждая вкладка могут выбирать свое юрлицо независимо.

## Проверка после обновления

1. Войти администратором и проверить раздел «Настройки».
2. Войти каждым сотрудником в отдельном профиле Chrome и убедиться, что чужие юрлица отсутствуют.
3. Открыть несколько вкладок WB Продвижения и проверить, что панель загружается во всех вкладках.
4. Обновить расширение на странице `chrome://extensions`, затем обновить уже открытые вкладки WB.
5. Выполнить ручное обновление WB и сверить одну кампанию по ID, бюджету, CTR и ДРР.
6. Проверить журнал без включения боевых операций.
7. После включения боевых операций проверить одну тестовую кампанию с небольшим лимитом.
