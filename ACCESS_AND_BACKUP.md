# Доступ сотрудников и резервные копии

## Доступ к юрлицам

Дополнительные пользователи по-прежнему задаются в `ADDITIONAL_USERS_JSON`. После первого запуска администратор может открыть **Настройки → Доступ сотрудников к кабинетам** и отметить юрлица для каждого сотрудника.

- Ограничение проверяется сервером, поэтому закрытый кабинет нельзя открыть прямой ссылкой или запросом.
- Новое юрлицо не добавляется автоматически сотруднику, если администратор уже сохранил для него список доступных кабинетов.
- Если список ещё не сохранялся на сайте, применяется `legal_entity_ids` из `ADDITIONAL_USERS_JSON`.
- Администратор видит все кабинеты.
- Если снять у сотрудника все флажки, он не увидит ни одного кабинета.

## Создание резервной копии

Из каталога проекта выполните:

```bash
cd /opt/wb-autofund
bash backup_server.sh
```

Скрипт создаёт согласованный снимок SQLite, проверяет его командой `PRAGMA integrity_check` и сохраняет файл в `/opt/wb-autofund/backups`. Исходный Docker volume не изменяется.

Проверка наличия volume:

```bash
docker volume ls | grep wb_autofund_data
```

Рекомендуется запускать копирование ежедневно через cron и дополнительно перед каждым обновлением. Храните вторую копию вне этого сервера.

## Восстановление

Сначала остановите приложение и сохраните текущую базу отдельным файлом. Затем замените `/data/wb-autofund.sqlite` в volume выбранной проверенной копией и снова запустите приложение. Перед восстановлением обязательно зафиксируйте имя файла резервной копии и имя volume:

```bash
cd /opt/wb-autofund
docker-compose down
docker volume ls | grep wb_autofund_data
ls -lh backups
```

После восстановления:

```bash
docker-compose up -d
docker-compose ps
curl -fsS http://127.0.0.1:4173/api/health
```
